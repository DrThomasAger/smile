What is (: Smile?

(: Smile is a minimal, human-readable markup language designed to make LLM prompts simple, structured, and maintainable. Use (:, [=, and similar emoji-based brackets to organize instructions for clear model response and increased instruction following.

Think of (: Smile like HTML, but for prompt instructions. Structure blocks for precise, repeatable outputs!

How to Use

  1. Start each section with (: and name it (e.g., (: Format ().
  2. End sections with closing brackets (e.g., ) End Format :)).
  3. Define clear separation between instructions, data, and response formats.
  4. Use special symbols for strict/literal instructions: [=, quotes [""], variables [$var$], importance [! ... !].
Smile syntax is flexible: add or remove structure as needed for the highest instruction following results.

FAQ

Why structure prompts?

Consistent, structured prompts lead to better model responses across teams and projects. Structure reduces drift, hallucination, and context bleed.

How is (: Smile different from Markdown?

Markdown is optimized for document rendering‚Äînot for LLM instruction clarity. (: Smile is designed strictly for instruction following and meta-delimitation within model prompts.
